# Definition of Done (ABD)

A task is DONE only when all conditions are met.

## Required Conditions

- Goal is explicitly defined
- Behavior specification exists
- Prompt was executed
- Output was reviewed
- Scorecard was filled
- Evidence exists or is justified
- Recycle output is recorded
- Prompt changes are logged if applied

## Invalid Done States

A task is NOT done if:
- output looks correct but behavior rules were violated
- code works but no evidence exists
- prompt failed and was not patched
- recycle output is missing
